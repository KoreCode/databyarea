# Leaflet + GeoJSON Setup Steps (State → County → City)

This guide shows how to add a hoverable, clickable county map with Leaflet and GeoJSON so users can navigate from state pages to county pages.

## 1) Pick your GeoJSON source

Use one of these datasets:
- US counties GeoJSON (with FIPS): suitable for county hover/click behavior.
- Optional state-specific subsets (smaller payload, faster pages).

Recommended structure in this repo:
- `assets/geo/us-counties.geojson` (full file), or
- `assets/geo/counties/<state-slug>.geojson` (per-state files).

## 2) Add county metadata for URL routing

Leaflet click handlers need a mapping from GeoJSON feature to your county page URL.

Create a generated JSON lookup keyed by state+county FIPS, for example:

```json
{
  "48-201": "/property-taxes/texas/harris-county/",
  "48-113": "/property-taxes/texas/dallas-county/"
}
```

Store as:
- `assets/geo/county-page-lookup.json`

## 3) Add Leaflet assets

In each state template/page where the map should render:

```html
<link
  rel="stylesheet"
  href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css"
  integrity="sha256-p4NxAoJBhIIN+hmNHrzRCf9tD/miZyoHS5obTRR9BMY="
  crossorigin=""
/>
```

Before `</body>`:

```html
<script
  src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"
  integrity="sha256-20nQCchB9co0qIjJZRGuk2/Z9VM+kNiyxNV1lvTlZBo="
  crossorigin=""
></script>
```

## 4) Add a map container

```html
<div class="card">
  <h2>Browse counties on a map</h2>
  <div id="countyMap" style="height: 460px; border-radius: 12px;"></div>
  <p class="mutedSmall">Hover to preview county names. Click a county to open its page.</p>
</div>
```

## 5) Initialize Leaflet and bind GeoJSON

```html
<script>
  const map = L.map('countyMap', { scrollWheelZoom: false }).setView([31.0, -99.0], 6); // Texas center example

  L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    maxZoom: 12,
    attribution: '&copy; OpenStreetMap contributors'
  }).addTo(map);

  const styleDefault = {
    color: '#334155',
    weight: 1,
    fillColor: '#93c5fd',
    fillOpacity: 0.45
  };

  const styleHover = {
    color: '#1d4ed8',
    weight: 2,
    fillColor: '#60a5fa',
    fillOpacity: 0.7
  };

  Promise.all([
    fetch('/assets/geo/counties/texas.geojson').then(r => r.json()),
    fetch('/assets/geo/county-page-lookup.json').then(r => r.json())
  ]).then(([geo, countyLookup]) => {
    const layer = L.geoJSON(geo, {
      style: styleDefault,
      onEachFeature: (feature, l) => {
        const countyName = feature.properties.NAME;
        const stateFips = feature.properties.STATEFP;
        const countyFips = feature.properties.COUNTYFP;
        const key = `${stateFips}-${countyFips}`;
        const href = countyLookup[key];

        l.bindTooltip(countyName, { sticky: true });

        l.on('mouseover', () => l.setStyle(styleHover));
        l.on('mouseout', () => layer.resetStyle(l));
        l.on('click', () => {
          if (href) window.location.href = href;
        });
      }
    }).addTo(map);

    map.fitBounds(layer.getBounds());
  });
</script>
```

## 6) Add a fallback county list for SEO and accessibility

Keep regular crawlable links below the map (already supported by county blocks) so bots and users without JS still navigate successfully.

## 7) Keep payloads small

For production:
- Prefer state-specific GeoJSON files over a nationwide file on each page.
- Simplify geometry (TopoJSON/Mapshaper) to reduce file size.
- Cache GeoJSON through CDN.

## 8) Where to integrate in this repo

Best insertion points:
- State pages generated by `publish_popular_cities_daily.py` (`inject_county_block` output area).
- State template in `scripts/ensure_states.py` so new state pages include map container/anchor by default.

## 9) Optional enhancements

- Color counties by a metric (property tax rate, utility cost index, etc.).
- Add legend and county search box.
- Add "View all cities in county" popup action.

## 10) QA checklist

- County hover tooltip appears correctly.
- Clicking counties navigates to the right county URL.
- Mobile interaction works (tap).
- Non-JS fallback list remains visible and functional.
- Lighthouse performance remains acceptable after map load.
